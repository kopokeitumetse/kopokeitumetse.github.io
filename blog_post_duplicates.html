The Problem

Duplicate files are a silent drain on storage. After inheriting IT environments from previous admins, I've found the same pattern repeatedly: a shared drive or user machine that's technically "almost full" turns out to have 8–15% of its storage consumed by exact duplicate files — same content, different names, scattered across different folders. Photos copied to multiple backup folders. Documents saved under slightly different names. Downloaded installers never cleared out.

This script does a full-system scan, identifies every set of duplicate files, and exports a clear CSV report showing exactly where each duplicate lives and how much space you'd recover by removing it. It never touches, moves, or deletes anything — the decisions are yours.

How It Works

Naively comparing every file against every other file is too slow for a full system scan. The script uses a two-phase approach to keep it practical:

  1. Size pre-filter: Files with unique sizes cannot possibly be duplicates. Only files sharing a size with at least one other file are sent to the hashing phase. On a typical system this eliminates 60–80% of files immediately.
  2. SHA256 hashing: Candidate files are hashed. Files with identical hashes are confirmed duplicates — same content byte for byte, regardless of filename.

Usage

Basic — scan all fixed drives

PS> .\Find-DuplicateFiles.ps1

Scan a specific folder only

PS> .\Find-DuplicateFiles.ps1 -ScanPath "C:\Users"

Focus on large files (minimum 50 MB)

PS> .\Find-DuplicateFiles.ps1 -MinSizeKB 51200

Save report to a custom location

PS> .\Find-DuplicateFiles.ps1 -OutputPath "C:\Reports\duplicates.csv"
Run as Administrator for full system access. Without elevation, files in system-protected directories will be skipped silently. System folders (Windows, Program Files, Temp) are excluded by default regardless.

Understanding the CSV Report

The output CSV has one row per duplicate file. Each set of duplicates shares a DuplicateGroup number, so you can sort or filter by group in Excel.

ColumnDescription
DuplicateGroupAll files with the same number are exact duplicates of each other
FileNameJust the filename
FullPathComplete path including drive — where the file actually lives
DriveWhich drive the file is on (C:, D:, etc.)
SizeMBFile size in megabytes
LastModifiedLast write timestamp — oldest is likely the original
CopiesInGroupTotal number of copies of this file found
WastedSpaceMBStorage you'd recover by keeping only one copy
SHA256HashThe file hash — all duplicates in a group share this value

Within each duplicate group, the oldest file is listed first. This is usually (not always) the original. Never delete files blindly based on the report — open both copies and verify before removing anything.
