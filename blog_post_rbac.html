Getting AWS Cloud Practitioner Certified: My Study Approach

2024-08-15|7 min read| tutorial aws certification

The AWS Cloud Practitioner (CLF-C02) is the entry point into the AWS certification path. It's not a technical deep-dive — there's no hands-on lab component and no architecture design. What it tests is whether you understand the AWS ecosystem well enough to make informed decisions: which services exist, what they do, how the pricing model works, and what the shared responsibility model means in practice. For an on-premise IT administrator moving toward cloud-adjacent roles, that framing is exactly right.

This is how I studied for and passed it, coming from a background in on-premise infrastructure with no prior AWS experience.

1. What the Exam Actually Tests

The CLF-C02 is split into four domains. Understanding the weight of each domain is the first step in allocating study time:

DomainWeightWhat It Covers
Cloud Concepts24%What cloud is, the six advantages of cloud, deployment models
Security & Compliance30%Shared responsibility, IAM, compliance programs, encryption
Cloud Technology34%Core services: EC2, S3, RDS, VPC, Lambda, Route 53, CloudFront
Billing & Pricing12%Pricing models, cost management tools, support plans

Cloud Technology and Security together make up 64% of the exam. Everything else supports those two sections.

2. Study Resources (What I Used)

AWS Skill Builder — Free Tier

AWS's own learning platform has a free Cloud Practitioner Essentials course that covers the full exam scope. It's dry but comprehensive. I used it for initial exposure to every domain — one module per day over two weeks. Do not skip the knowledge checks at the end of each module; they're better exam preparation than they look.

Practice Exams

The most important resource by far. I used two sets:

  • AWS Official Practice Question Set (free on Skill Builder) — 20 questions, explains every answer. Do this first.
  • Tutorials Dojo CLF-C02 Practice Exams — 6 full practice exams, 65 questions each. Harder than the real exam, which is the point.

I ran practice exams until I was consistently scoring above 85%. The real exam felt easier after that.

3. The Services You Must Know

The exam does not test every AWS service in depth, but it does expect you to recognise what the core services are for and roughly when you would use each one. These are the ones that appeared most frequently:

4. The Shared Responsibility Model — Learn It Cold

This concept appears in multiple questions across Security and Cloud Concepts. The model is simple in principle: AWS is responsible for security of the cloud (physical infrastructure, hypervisors, hardware), and you are responsible for security in the cloud (your data, your OS configuration, your IAM policies, your application security).

"AWS patches the hypervisor. You patch your EC2 instances."

Where it gets tricky in exam questions is at the service boundary. RDS is a managed service — AWS patches the database engine. EC2 is not managed — you patch the OS and any software running on it. Lambda is fully managed — you're only responsible for your function code. Know which services shift responsibility to AWS and which leave it with you.

5. My Study Timeline

  1. Week 1–2: AWS Skill Builder course, one module daily. Notes on each domain.
  2. Week 3: Official practice question set. Identify weak domains (mine was Billing).
  3. Week 4: Tutorials Dojo practice exams — two full exams, review every wrong answer.
  4. Week 5: Two more practice exams. Read the AWS whitepapers for shared responsibility and pricing.
  5. Exam day: Scheduled morning slot. Passed.
For on-prem admins: The cloud concepts feel abstract until you map them to things you already know. IAM is Active Directory. VPC is your LAN. Security Groups are your host-based firewall. Once the translation clicks, the rest follows quickly.

What Comes Next

Cloud Practitioner is the foundation, not the destination. My next step is AWS Solutions Architect Associate (SAA-C03), which moves from conceptual understanding into actual architecture — designing multi-tier applications, choosing the right storage class, building resilient systems. That's where the on-prem to cloud translation becomes genuinely useful.
